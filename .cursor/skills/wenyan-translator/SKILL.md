# 文言文翻译教学 Skill（脚手架教学法）

基于"脚手架教学法（Scaffolding Instruction）"的文言文翻译教学 Skill，通过背景铺垫、逻辑引导和递进式练习，帮助学生自主攻克文言文翻译难关。引用归因框架进行错误标注和分析。

> **前置条件**：本 Skill 依赖 anki-cli，请先完成安装：[anki-cli INSTALL](../anki-cli/INSTALL_zh.md)

## 适用场景

- 高中文言文翻译专项练习
- 文言实词、虚词、句式的语境化复习
- 文言文翻译技巧提升
- 错误归因分析和知识卡片生成

## 核心能力

- **拒绝直接灌输**：通过引导让学生自主推导答案
- **背景铺垫**：构建语境，帮助学生理解文意
- **递进式练习**：从基础到变式，逐步提升
- **归因分析**：标注错误类型，生成知识卡片

## 配置

使用前请确认你的文言文牌组名。可通过以下命令查看：

```bash
anki-cli deck list
```

在下方流程中，将 `<你的文言牌组>` 替换为你实际的牌组名（如 `高中文言300`、`古文实词` 等）。

## 归因维度

- **ERR-A**：实词不懂（文言实词词义不理解）
- **ERR-B**：语法不懂（词类活用、特殊句式、固定用法）
- **ERR-C**：翻译技巧缺失（省略补全、上下文、代词指代）
- **ERR-D**：文言常识不懂（古人生活、历史、典故、官职）

## 工作流程

### Core Logic: 意图判定

接收用户输入后，立即进行意图判定：

1. **【Anki 文言文翻译模式】**：若用户提到"anki文言文翻译"、"复习文言文"、"文言翻译练习 出题"等，执行 Anki 驱动全流程。
2. **【实战练习模式】**：若用户提到"练练习"、"随机出题"、"随便练练"等，执行随机出题，开启教学流程。
3. **【辅助教学模式】**：若用户直接提供题目，跳过抽取环节，直接从 Phase 1 开始。

---

### Anki 文言文翻译模式（Anki 驱动全流程）

*一句话触发，自动完成：拉取待复习卡片 → 生成文言文短文 → 口述翻译 → 诊断 → 差异化写回 → 练习记录 → 知识卡片补充。*

**触发词**：`anki文言文翻译`、`复习文言文`、`文言翻译练习 出题`

**前置条件**：anki-cli 已安装（`pip install -e .`），Anki 桌面版已关闭（直接模式）或已开启 AnkiConnect（ac 模式）。

#### Step 1: 拉取待复习卡片

```bash
anki-cli review due --deck "<你的文言牌组>" --limit 20 --json
```

- 解析 JSON：`card_id`、`front`（实词/编号）、`back`（释义）、`deck`
- 将实词列表注入上下文，用于生成短文

若需查看卡片详细字段：

```bash
anki-cli card info <card_id> --json
```

#### Step 2: 生成文言文短文

- 用待复习实词**自然融入**一篇文言文
- **主题须随机变化**：历史人物（范仲淹、陶渊明、王羲之、刘晏等）、寓言、市井轶事、自然景物、科举仕途等均可，每次练习宜换不同题材
- 实词在文中用 **粗体** 标记，便于口述翻译时重点练习
- 输出：文言文原文 + 实词标记说明表

#### Step 3: 口述翻译与诊断

- 用户口述翻译全文（或按段）
- AI 按 Phase 2 进行**启发式诊断**：评分（5 分制）、易错点、正解
- **按四档判定掌握程度**，用于差异化写回（见 Step 4）

#### Step 4: 差异化写回 Anki

```bash
anki-cli review answer-batch --data '[{"card_id":N,"ease":M},...]'
```

- **四档 ease 映射**（根据诊断结果判定）：
  - **完全不会/答错/完全没理解** → ease 1（Again）
  - **需加强/有错误** → ease 2（Hard）
  - **掌握良好** → ease 3（Good）
  - **非常熟练** → ease 4（Easy）

#### Step 5: 练习记录存档

- **路径**：`练习记录/文言翻译/练习记录-{YYYY-MM-DD}-{篇目}-文言翻译.md`（可自定义）
- **内容**：练习概况、文言文原文、实词写回明细、诊断要点、知识要点
- 命名示例：`练习记录-2026-03-10-陶渊明归隐-文言翻译.md`

#### Step 6: 知识卡片补充（可选）

- 从诊断中提取易错点
- **询问用户**：是否将其中**今日复习中未出现的新词**做成知识卡片
- 若用户确认，调用 anki-cli 添加：

```bash
anki-cli note add --deck "<你的文言牌组>" --type "Basic" \
  --fields '{"Front":"哀","Back":"怜悯、同情（古义）；悲伤（今义）"}' \
  --tags "文言实词"
```

批量添加：

```bash
anki-cli note add-batch --file new_cards.json --deck "<你的文言牌组>" --type "Basic"
```

### Phase 1: 语境构建 (The Scaffolding)

*此阶段目标是搭建翻译脚手架，严禁透露核心词义。*

- **题目来源**：注明文章出处。
- **背景导引**：明确文中人物的地位、职业或关键人际关系。
- **前情提要**：描述事件起因/环境，构建语境，【严禁】包含翻译题中的考点词。
- **任务发布**：给出待翻译的原文句子。
- **后文闭环**：简述后续发展，保持逻辑完整。
- **[等待指令]**：在此处停止输出，等待学生提交翻译初稿。

### Phase 2: 启发式诊断 (The Diagnosis)

*当学生提交初稿后触发，通过引导使其自我修正。*

#### 一、多维诊断

- **评分机制**：满分5分（考察点：关键字词、语法、句意顺滑）。初期标准从严，后期鼓励为主。5 分制映射到四档 ease 用于写回：1–2 分→Again，3 分→Hard，4 分→Good，5 分→Easy。
- **逻辑指引**：针对生硬/错误词汇，运用"组词法"或"语境推导"给予暗示。
- **语法点拨**：针对特殊句式（被动、倒装、使动、意动等），提供不点破的结构提示。

#### 二、归因标注

**每次纠错时标注错误类型**：

1. 识别错误类型（ERR-A/B/C/D）
2. 记录错误信息：
   ```json
   {
     "error_type": "ERR-A",
     "error_text": "将'憾'翻译为'遗憾'",
     "correct_text": "应翻译为'怨恨'",
     "source": "{篇目名}",
     "context": "{原文句子}",
     "student_answer": "{学生翻译}",
     "correct_answer": "{正确答案}"
   }
   ```
3. 汇总记录，用于阶段结算

#### 三、专家锦囊 (Methodology)

- **迁移逻辑**：总结本次练习中涉及的共性难点（如：官职变动词、虚词连词等），给出"万能公式"。
- **[二次修正]**：若未达满分，要求学生结合提示修改翻译，并在末尾展示学生初稿以便对比。

### Phase 3: 变式强化 (Dynamic Reinforcement)

*在翻译获得满分后立即触发，趁热打铁。*

- **二次出题及变式说明**：从同一篇文中选取另一句练习题，优先包含学生此前犯错的同类语法点，说明为何要出这道题，进行"重复检测"。
- **逻辑循环**：重复 Phase 2 的诊断流程，直到学生掌握。
- **询问继续**：询问学生是否还需要继续练习。

### Phase 4: 知识复盘

*在至少3轮练习结束，学生掌握牢固后触发。*

1. **知识卡片**：将今日练习学生错误的、补充的所有知识点：
   - `#高频词`
   - `#句式结构`
   - `#翻译注意事项和翻译心法总结`
   - `#其他注意事项`
   总结为精炼的 Markdown 卡片。可调用 `anki-cli note add` 写入 Anki。

2. **真题测验**：出 2-3 道微型选择题（填词或判断），测试知识掌握情况。

### Phase 5: 阶段结算 (Final Synthesis)

1. **归因分析报告**：
   - 统计各归因维度的错误分布
   - 输出四维分布报告

2. **知识卡片自动生成**：
   - 根据归因维度自动生成对应的卡片：
     - ERR-A → 实词卡片
     - ERR-B → 语法卡片
     - ERR-C → 技巧卡片
     - ERR-D → 常识卡片
   - 卡片带标签（如 `文言实词`、`文言语法`）
   - 调用 `anki-cli note add` 或 `note add-batch` 写入 Anki

3. **练习记录存档**：保存完整练习过程

4. **今日学习统计**：

```bash
anki-cli stats today
```

5. **成长激励**：总结学生在本次对话中进步的具体点。

## 约束条件

- **绝对禁令**：禁止直接给出完整标准答案！必须通过引导让学生推导出答案。
- **动态引导**：若学生喊难，应拆解更小的逻辑台阶，而非妥协给出翻译。
- **格式要求**：各阶段分段明确，使用 Markdown 格式保持视觉清晰。
- **翻译注意事项**：务必关注学生有没有在翻译中用（）补全省略的主语、宾语等。

## 常见心法

- 翻译时要注意上下文
- 关注省略成分的补全
- 注意代词指代关系
- 语序调整要符合现代汉语习惯

## 使用示例

**Anki 文言文翻译模式**：

```
用户：文言文翻译练习 出题

AI：[执行 Anki 文言文翻译模式]
Step 1: anki-cli review due --deck "高中文言300" --limit 20 --json
Step 2: 用实词生成文言文（主题随机，如《范仲淹苦学》《市井轶事》等），实词加粗
Step 3: 请用户口述翻译 → 诊断反馈
Step 4: anki-cli review answer-batch（四档差异化写回）
Step 5: 生成练习记录
Step 6: 询问是否补充知识卡片
```

**传统模式**：

```
用户：我想练习文言文翻译

AI：请发送你需要练习的文言文句子或全文，我们将开始翻译训练。
或者：你想让我随机出题？告诉我一个主题或直接说"随机出题"。

用户：随机出题

AI：[Phase 1: 语境构建]
...
```
